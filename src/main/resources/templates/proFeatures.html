Promote Topics

Topics can be promoted from one environment to another (lower to higher). With approvals.

Delete Topics

Topics can be deleted from a selected environment. With approvals.

Naming conventions for new Topics

Topics can be created prefixed with a name if there is a prefix configured on an environment.
Ex: "dev" is a prefix configured when creating an environment. When reuesting for topic in dev environment , it forces users to prefix "dev".

Prefixed Acls

Prefixed access controls can be requested

TransactionalID (idem potency) Acls

TransactionalID access controls can be requested

Delete Acls

Acls can be deleted with approvals

View Schemas

Avro schemas (latest version) of all subjects on all environments can be viewed.

Synchronize/Backup of Configuration

All the existing topics and access controls on Kafka cluster can be associated to Teams and hence stored in metastore.
With this feature, you can already start using Kafkawize, with ease.

Restore of Configuration

All the existing topics and access controls on Kafkawize metastore can be created back on any selected Kafka cluster. (In case there is a crash of ZK/Broker)
In no time, a copy of a cluster is created.
Audit of the whole activity.

Active Directory integration

Kafkawize can be integrated into your organization's existing Active directory, so users do not have to enter or create a new password to login to Kafkawize.

Ldap server integration

Kafkawize can be integrated into your organization's any existing Ldap server, so users do not have to enter or create a new password to login to Kafkawize.

Multi Tenancy

Each tenant can manage their topics with their own teams in isolation. Every tenant can have their own set of Kafka environments, and users of one tenant cannot view/access topics, acls or any from other tenants.
It provides an isolation avoiding any security breach.

Analytics

Several charts are introduced to give an overview of Clusters and usages by teams.
Topics, Acls and Partitions per team
Topics, Acls and Partitions per environments
Activity log per team and per environments

Reports

A full data report can be downloaded for further analysis

Configurable Roles

Any new roles can be added and associate different kind of permissions

Permissions

A whole bunch of permissions can be assigned to roles from User interface, making it very flexible. With immediate effect, users will be able to see the changes.
Ex: A permission to request topics, or approve or add users, environments, clusters etc.

Email notifications

Users receive email notifications on every request, on approvals, new user creations, etc

Kafka connectivity

ClusterManagement api can connect to Kafka on various protocols/mechanisms like SSL, SASL_SSL, SASL_PLAINTEXT and GSSAPI (Kerberos)

Support

Kafkawize provides support in white labelling, company colors, installation and new features
