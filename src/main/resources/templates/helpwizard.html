Configure Klaw Wizard

Welcome

Congratulations !!


Klaw can be configured easily if you follow the next steps from here

In these steps we are going to configure kafka clusters, environments, teams, users, roles and permissions
For any questions or support, please click the Help button on the top right.

Configure Kafka Clusters

Configure Kafka clusters !!


Configure your kafka clusters from here. You can add any number of clusters. Only users with SUPERADMIN roles can add.

After adding the cluster, if you have selected SSL as the protocol, download the public key and import to your kafka broker certificates.

Execute the command provided which authorizes Klaw to create Kafka Topis, subscriptions (acls), etc
Configure Environments

Configure Environments !!


Environments are wrappers over your kafka clusters. You can have multiple environments connected to one Kafka cluster with different settings

Configure your environments from here. You can add any number of environments. Only users with SUPERADMIN roles can add by default.

You can configure min max partitions, replication factors here.

Configure Environments hierarchy

Configure Environments hierarchy !!


klaw.tenant.config has to be configured for promotion of topics, or to keep a base environment to be a low level environment.

Configure this from here. Only users with SUPERADMIN roles can add by default.

Test Kafka connectivity

Test Kafka connectivity !!


After configuring kafka environments,
1. Upload the public key of your broker keystore
2. Import public key of Klaw to your truststore
3. Execute provided acl command.
Now you can test the connectivity with the button provided at here.

Synchronize Topics and Acls

Synchronize Topics and Acls !!


If you already have topics on your brokers, you can synchronize the topics and Producer and Consumer acls with Klaw.

Synchronize Topics from here.

Synchronize Acls from here.

Create Teams

Create Teams !!


After adding clusters and environments, now you can create Teams.

Create a team from here. Only users with SUPERADMIN roles can add by default.

Create Users

Create Users !!


After adding teams, now you can create users and associate to teams.

Create a user from here. Only users with SUPERADMIN roles can add by default.

Optional Roles and Permissions

Create Roles and Permissions !!


You can create roles and assign permissions to those roles if you need so.

Create a role from here. Only users with SUPERADMIN roles can add by default.

Associate permissions to roles from here.

You are done. Hurray !!

You are done. Hurray !!


Your teams can now request for topics, acls, connectors and schemas.

To test this, you can request for a topic from here. Make sure your role has permission to request/approve topics.
