Topics

After selecting topics all the topics are displayed from all Kafka environments.
You can search for topics with topic name or selecting a team., or can be searched based on the available documentation.

If you do not see the topics, there could be multiple reasons. No topics have been requested through Kafkawize, or no topics are synchronized, or there is a database connection issue. Please check with your administrator for Help from tp right.

Topic Overview


To see the details of a topic, click on any of the topics from topics page. Topic details are shown.
In topic details, all topic partitions and replication factor per each environment are shown.
At the bottom of the topic overview page, all the subscriptions (acls are shown).

New topic

To request for a new topic Request Topic
Note : You should have REQUEST_CREATE_TOPICS permission to request for a topic.
The environments displayed here are being retrieved from team configuration, which is a subset of kafkawize.tenant.config property. A full list of request topics is configured at Server configuration for property kafkawize.tenant.config

Promote Topic

If a topic exists in lower environment, it can be requested for promotion to next environment if your team is the owner of the topic from overview page.
It is possible to select topic partitions from this page specific to that target environment.
On submission, a request is generated for approval who has APPROVE_TOPICS permission. After approval, topic is promoted.

Delete Topic

From the topic overview page, a topic can be submitted for deletion. You can submit a deletion only if your team is the owner of the topic and there are no subscriptions(producers and consumers) for the topic.
On submission, a request is generated for approval who has APPROVE_TOPICS permission. After approval, topic is deleted from the environment.
If a topic exists in Dev, Tst, Acc and Prod environments, it is possible to delete the topic in Prod first.

Claim Topic

Teams can gain ownership which they do not own. In this case, a Topic claim request is raised and should be approved by exisiting owners.
On approval, topic ownership is updated with new Team owner.

Topic promotion environment hierarchy

From Server configuration, environment hierarchy can be configured. For ex: DEV,TST,ACC,PRD

Approve/Decline Topic requests

Users who have APPROVE_TOPICS permission can approve/decline topic requests from Approve Topics page.
With this permission APPROVE_ALL_REQUESTS_TEAMS, it is possible to approve decline all requests of all teams within same tenant.

Access Controls

After a user selects a topic from topics, he/she will be in the TopicOverview page.
At the bottom, producer/consumer subscriptions are visible.
Prefixed acls and TransactionId acls are visible in different tabs

Subscribe to a topic

In the topic overview page, there is a Subscribe button which opens a page to request for a new subscription.
After selecting an environment, you can select Producer or Consumer.
If you select Producer, you would see two options. You can choose between LITERAL or PREFIXED. Optionally you can mention a transactionalId.
Note:For prefixed acls, it is mandatory to have atleast one topic existing in Kafkawize with that prefix.
If you select Consumer, consumer group should be mentioned.
Acl can be of IP address or Principle based.

Note: Only users with REQUEST_CREATE_SUBSCRIPTIONS can submit new subscription requests.
After submission, request is generated and sent to Topic owner team for approval, and an email notification is sent.
Approver needs APPROVE_SUBSCRIPTIONS permission.

Approve/Decline Acl requests

Users who have APPROVE_SUBSCRIPTIONS permission can approve/decline acl requests from Approve Acls page.

Avro Schemas

We are referring to Avro schemas which can be registered in Confluent Schema registry.
At this moment, only the latest schemas for a subject (topic) in each environment on schema registry are visible in Schemas tab in TopicOverview page.

Request for schema

To submit a new schema request go to Request Schema
After selecting an environment, and topic, copy paste the Avro schema in the textarea of Schema.

Note:Your team should be the owner of the topic to request for a schema.
You should have REQUEST_CREATE_SCHEMAS permission to request for a schema.
After submission, a request is generated and sent to your team for approval. Approvers need APPROVE_SCHEMAS permission.

Approve/Decline Schema requests

Users who have APPROVE_SCHEMAS permission can approve/decline schema requests from Approve Schemas page.

Analytics

Analytics at a tenant level can be seen from this page
- Total topics,partitions,subscriptions per environment.
- Total topics per team in all clusters
- Producer and Consumer subscriptions per team can be seen
- Total requests per day, which includes topic and subscription requests for the last 30 days.

Download report

An excel report can be downloaded which includes all the above statistics. Topics per environment are part of the report.
Report varies based on roles., and which team you belong to.

Dashboard

Dashboard is here. Lot of information is available here.
- Topics owned by my team, and can be viewed
- My team members
- Topics to which my team applications are producing to.
- Topics from which my team applications are consuming from.
- A graph displaying the activity log of your team.
- A graph displaying total topics per cluster which are owned by your team.
- Pending topic/subscription/schema requests
- All Kafka and schema environments and their online status

Download report

An excel report can be downloaded which includes all the above statistics.

Synchronize From Cluster

Synchronize is available in the main menu. Topics and Subscriptions from an existing Kafka environment can be associated to teams.

Topics from cluster

From here, you can retrieve all the topics from selected Kafka environment (connected Kafka cluster).
You will see a list of topics from that environment. You can associate a team to each of those topics.
If a topic exists in multiple environments, it is mandatory to assign a team to a topic from Base Sync Cluster
Note:It is not possible to assign different teams to same topic which exists in different environments.
A user with permission SYNC_TOPICS can perform this action.

Acls from cluster

From here, you can retrieve all the subscriptions (LITERAL) from selected Kafka environment.
You will see a list of producer and consumer subscriptions from that environment. You can associate a team to each of them.
A user with permission SYNC_SUBSCRIPTIONS can perform this action.

It is possible to change teams for any topics at any point of time from this feature.

Synchronize To Cluster

Synchronize is available in the main menu. Topics and Subscriptions can be recreated back on Kafka cluster with data available on Kafkawize.
If you have any issues with Kafka cluster, and lost all your topics and acls, you can recreate all of them with one single click from here.

Topics to cluster

From here, you can retrieve all the topics from selected Kafka environment from within Kafkawize (not Kafka clsuter).
After selecting and environment, topics from that environment availablein Kafkawize are displayed. You can select a few or there is an option to select all the topics of source environment.
You need select a target environment and hit the Create button, which will start to create topics on the target environment.
A full log is displayed at the bottom of the page.

Server Configuration



The following can be configured from Server configuration
- Email Notifications content for all notifications
- Cluster Api connection parameters. Connection can be tested.
- Temporay location for downloading reports Ex : /tmp
- Email id of Super admin (kafkawize.superuser.mailid), for kafkawize users to contact in caes of any issues
- Retrieve Avro schemas (kafkawize.getschemas.enable) true/false
- Names of environments (kafkawize.envs.standardnames) DEV/TST..
- Allowed roles when adding a new user
- Tenant configuration (kafkawize.tenant.config). Base Sync cluster, order of topics promotion environments, request topics environments User who has UPDATE_SERVERCONFIG permission can update server configuration.

Cache reset

When a new property or server configuration is added or updated directly from Database, you can reset the cache from this page, to reflect the changes.

Stop Kafkawize

Kafkawize can be stopped from this page. User who has SHUTDOWN_KAFKAWIZE permission can stop.

Tenants

Tenants are introduce isolation of topics for all teams, and environments. Tenants are associated with teams and environments. Basically it provides an isolated environment.
Tenants can be accessed from Tenants page. A new tenant can be added from Add Tenant
Only users with ADD_EDIT_DELETE_TENANTS permission can add tenants.

Example


Tenant1 consists of DEV, TST, ACC, PRD, and teams Octopus, Starfish
Tenant2 consists of DEV, TST, ACC, PRD, and teams Warriors and Rebels
Now, Octopus and Starfish teams cannot see or request for any topics related to Tenant2 environments., and other way round too.
By default, one tenant is always installed.

Clusters

Clusters define the bootstrap servers and their protocols
Clusters can be accessed from Clusters page. A new cluster can be added from Add Cluster
Only users with ADD_EDIT_DELETE_CLUSTERS permission, can add, modify, delete clusters.

Environments

Environments are associated with clusters and tenants., and other parameters like default partitions, max partitions, replication factor, and topic prefix and suffix.
Environments can be accessed from Environments page. A new tenant can be added from Add Environment
Only users with ADD_EDIT_DELETE_ENVS permission, can add, modify, delete Environments.

Teams

Teams are associated with tenants. There is an option to select to which environments, topics can be requested by this team.
By default all available environments defined in ServerConfig tenant configuration, are applicable.
Teams can be accessed from Teams page. A new team can be added from Add Team
Only users with ADD_EDIT_DELETE_TEAMS permission, can add, modify, delete Teams.

Users

Users are associated with teams., and roles.One user can have only one role.
Users can be accessed from Users page. A new user can be added directly by user from Add User by user who has ADD_EDIT_DELETE_USERS permission.

New User

A new user can register from Register page. After submitting request, it has to be approved.
Only users with ADD_EDIT_DELETE_USERS permission, can approve a new user request.
In case of Active directory, approved users, can use their AD account password.

Approve/Decline New User requests

Users who have ADD_EDIT_DELETE_USERS permission can approve/decline user requests from Approve Users page.

Roles

Every user has a role. All roles can be accessed from Roles page.
A new role can be added by user who has from ADD_EDIT_DELETE_ROLES permission., from page Add Role
